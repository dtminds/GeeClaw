# Support

This document explains where to ask for help and where to report different kinds of problems in GeeClaw.

## Where To Go

- Bug reports: open a GitHub issue with the bug report template.
- Feature ideas: open a GitHub issue with the feature request template.
- Documentation fixes: open a GitHub issue or pull request.
- Security issues: follow [SECURITY.md](SECURITY.md) and do not post the details publicly first.

## What To Include

Please try to include:

- app version or commit
- operating system
- whether you are using a packaged build or `pnpm dev`
- clear reproduction steps
- screenshots, logs, or short recordings when helpful

## Support Expectations

GeeClaw is currently maintained on a best-effort basis.

- We may not be able to answer every setup or environment-specific question immediately.
- Reproducible issues with clear steps are much easier to act on than broad descriptions.
- Pull requests that include tests or documentation updates are especially helpful.

## Before Opening An Issue

- Read [README.md](README.md) for setup and runtime notes.
- Read [CONTRIBUTING.md](CONTRIBUTING.md) for development workflow and guardrails.
- Check whether the issue is already described in an open issue or recent commit.

## Commercial Or SLA Support

There is currently no separate commercial support channel or guaranteed response SLA documented for this repository.
